Creating a MiniDAPP is simple.

They are standalone HTML5 sites, that run in full from the Minima application - not a website, using normal HTML, JavaScript and CSS and communicate with the Minima application over simple JSON RPC calls.

All the Minima logic to use the RPC is in a sinlge JS file.. minima.js.

A MiniDAPP is the folder with your html app, a minidapp.conf file and a couple of images for the 'Menu'.. all zipped up and renamed *.minidapp. You can open and inspect any MiniDAPP by using any regular archive program or unzip directly from the command line.


We recommend using the excellent decimal.js class for high precision mathematics built in - a MUST when using numbers in Minima as JavaScript default number handling is frankly embarrassing. Check the demos to see usage or look at the full versions in the js folder.

The Main MINIDAPP Hub is accessed as a website hosted directly in the Minima application..

Use http://127.0.0.1:9004/ if you are running the java locally or use the Phones' IP instead to browse from your desktop.

( SSL support coming.. everything still very much pre-ALPHA )

Running a node on the main testnet..

java -jar minima.jar

Running a private testnet is very simple. Use -clean to reset.

java -jar minima.jar -private -clean
