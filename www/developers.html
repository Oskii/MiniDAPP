Creating a MiniDAPP is simple.

They are standalone HTML5 sites, that run in full from any folder - not a website, using normal HTML, JavaScript and CSS and communicate with the Minima application over simple JSON RPC calls.

All the Minima logic to use the RPC is in a sinlge JS file.. minima.js.

A MiniDAPP is the folder with your html app, a minidapp.conf file and a couple of images for the 'Menu'.. all zipped up and renamed *.minidapp. You can open and inspect any MiniDAPP by using any regular archive program or unzip directly from the command line.

The MiFi demo shows how to integrate Minima into a normal website. The code is exactly the same as a MiniDAPP - and all you need to do is log into the Minima Pop Up that appears. Obviously - Users must be running Minima. We don't want another MetaMask on our hands..

The only change between using the MiniDAPP and MiFi minima.js file is MINIMA_IS_MINIDAPP variable. This stops the Minima PopUp from appearing.. and minima.js is served and editted automatically from the Minima app when using the MiniDAPP Hub.

The minima.js lib also come with the Decimal JS class for high precision mathematics built in - a MUST when using numbers in Minima as JavaScript default number handling is frankly embarrassing. Also a QR Code generator lib. Check the demos to see usage or look at the full versions in the js folder.

The Main MINIDAPP Hub is accessed as a website hosted directly in the Minima application..

Use http://127.0.0.1:21000/ if you are running the java locally or use the Phones' IP instead to browse from your desktop.
